# Skip-Bo (skip-bo)

부부 둘만 쓰는 무료 온라인 Skip-Bo 카드 게임. **운영비 0원이 절대 제약.**

게임 규칙은 `skipbo_rules_summary.md` 참조 (Stock/Hand/Building/Discard Pile, 1→12 순서 쌓기,
Skip-Bo = 와일드, 차례 끝낼 때 Discard 1장, Stock Pile 먼저 비우면 승리).

## 스택

| 영역 | 선택 | 호스팅(무료) |
|------|------|------|
| Frontend | React + Vite + TypeScript + PWA | Vercel |
| Backend | Node.js + Express + Socket.IO | Render Free |
| 영구 저장 | Turso (libSQL, SQLite 호환) | Turso Free |

### 왜 파일 SQLite가 아니라 Turso인가
Render 무료 인스턴스는 디스크가 **ephemeral** — 재배포/재시작 시 파일 SQLite가 삭제됨.
영구 디스크는 유료. 매치 누적 점수처럼 살아남아야 할 데이터는 Turso(무료·영구·SQLite 문법)에 둔다.

### 무료 티어 주의점
- **Render 콜드 스타트**: 15분 유휴 시 슬립 → 첫 접속 ~50초. 게임 진행 중엔 소켓 연결이 살아있어
  슬립하지 않으므로 실사용 영향은 첫 접속뿐. keep-alive 핑으로 우회하지 말 것(무료 한도 낭비).
- **2인 전용**: 인증·매치메이킹·로비 만들지 말 것. 고정 방 하나 또는 단순 공유 링크로 충분.

## 아키텍처 원칙

- **게임 상태(진행 중)는 서버 메모리**에 둔다. 활성 소켓 연결 중에는 인스턴스가 죽지 않음.
- **DB에 넣는 것은 매치 누적 점수(500점제)뿐.** 손패/빌딩 더미 등 진행 상태는 DB에 넣지 않는다.
- **규칙 검증은 전부 서버에서.** 클라이언트는 표시·입력만. 합법 수 판정(빌딩 순서, 와일드, Discard top)을
  믿을 만한 한 곳(서버 reducer)에 모은다.
- Socket.IO 이벤트는 `state` 단일 스냅샷 브로드캐스트로 시작. 델타 최적화는 둘이 쓰는 규모에선 불필요.

## 디렉터리

루트가 곧 client (Vite 앱). 별도 모노레포 도구 없이 `server/`만 추가했다.

```
src/          React 클라이언트 (루트 Vite 앱)
server/src/   Express + Socket.IO (별도 패키지)
shared/src/   engine.ts = 순수 게임 엔진 + 타입, client·server가 상대경로로 공유
```

- `shared/src/engine.ts`가 단일 진실. `apply()`가 권위 있는 reducer, `view()`가 좌석별 가림 상태.
- 패키지 매니저는 **pnpm** (lock 파일 존재).

## 개발 커맨드

터미널 2개로 띄운다.

```
# 1) 백엔드
cd server && pnpm install && pnpm dev      # http://localhost:3001

# 2) 프론트 (루트)
pnpm dev                                    # http://localhost:5173
```

- 엔진 자가검증: `pnpm dlx tsx shared/src/demo.ts` → `engine demo ok`
- 둘이 같은 PC/LAN에서 테스트: 브라우저 탭 2개로 각각 입장.

## 배포

- Vercel: `client/` 정적 빌드 자동 배포.
- Render: `server/` Web Service. 환경변수에 Turso `DATABASE_URL` / `AUTH_TOKEN`.
- CORS/소켓 origin은 Vercel 도메인으로 고정.
- **PWA**: MVP에선 미적용(게임 동작에 불필요). 배포 단계에서 `vite-plugin-pwa` 추가.
